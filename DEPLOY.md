# Инструкция по деплою на GitHub Pages

## Автоматический деплой (рекомендуется)

1. **Создайте репозиторий на GitHub** (если еще не создан)

2. **Измените base path в `vite.config.ts`**:
   - Если ваш репозиторий называется `smeta-app`, оставьте как есть
   - Если другое название, замените `/smeta-app/` на `/ваше-название-репозитория/`
   - Или используйте переменную окружения: `VITE_BASE_PATH=/ваше-название/`

3. **Закоммитьте и запушьте код**:
   ```bash
   git add .
   git commit -m "Initial commit"
   git push origin main
   ```

4. **Включите GitHub Pages**:
   - Перейдите в Settings → Pages вашего репозитория
   - В разделе "Source" выберите "GitHub Actions"
   - Сохраните

5. **Готово!** При каждом push в ветку `main` или `master` будет автоматический деплой.

## Ручной деплой

Если хотите задеплоить вручную:

```bash
# 1. Соберите проект
npm run build

# 2. Установите gh-pages (если еще не установлен)
npm install --save-dev gh-pages

# 3. Добавьте в package.json:
# "scripts": {
#   "deploy": "npm run build && gh-pages -d dist"
# }

# 4. Задеплойте
npm run deploy
```

## Настройка для собственного домена

Если у вас есть собственный домен:

1. В `vite.config.ts` измените `base` на `/` (корень)
2. В настройках GitHub Pages укажите ваш домен
3. Настройте DNS записи согласно инструкциям GitHub

## Важные замечания

- **HashRouter**: Приложение использует HashRouter для совместимости с GitHub Pages
- **Base path**: Убедитесь, что base path соответствует названию репозитория
- **Автоматический деплой**: Работает только для веток `main` или `master`
- **Данные**: localStorage работает на GitHub Pages, данные сохраняются локально у каждого пользователя
